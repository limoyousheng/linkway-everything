# 链接跳转 移动化设计

**日期**：2026-06-11
**目标**：在保留现有桌面端全部功能和数据的前提下，为 ≤ 1024px 设备新增一套手机/平板界面
**状态**：设计稿，待用户 review

---

## Context

`链接跳转.html` 当前是一份单文件 HTML 仪表盘（153 KB / 2821 行），功能完整但**只在桌面布局下可用**：浮动侧边栏 + 多列玻璃卡片网格 + 顶部 header 按钮组。在手机/平板上布局挤、按钮点不到、长按拖拽区域太小，无法日常使用。

`stitch_sleek_flow_navigation` 文件夹提供了一个 375px 宽的移动端参考模板（玻璃质感、Material 3 调色板、底部 tab 导航、Spotlight 搜索）。用户希望以此为蓝本叠加手机版，**桌面端功能与数据零改动**。

**已备份**：`链接跳转.pre-mobile-redesign-2026-06-11.html.bak`（153 KB，git 已忽略）

---

## 目标

1. ≤ 1024px 设备进入手机版界面（顶部栏 + 底部 tab + 单列卡片 + 底部 sheet 弹窗）
2. ≥ 1024px 设备保留现有桌面版界面
3. 数据层（localStorage + state-server）零迁移、用户已有数据无缝兼容
4. 桌面端全部功能（添加/编辑/删除卡、分类管理、主题切换、搜索、跨分类移动、SortableJS 长按拖拽、图检索 4-case 行为、状态同步）在手机端**功能等价**
5. 跨断点 resize 不闪烁、不丢数据、不重建 SortableJS 状态

## 非目标

- 不引入任何新依赖（不安装 npm 包 / 不引新 CDN）
- 不拆分多文件（保持单文件 HTML）
- 不重构现有桌面端布局
- 不动 state-server.py / 链接跳转.cmd / 默认 13 卡 / 默认 8 分类色 key

---

## 设计决策

### 决策 1：架构与数据流

**两套 DOM 树 + 单一数据层**

- 桌面端 `<div class="desk-*">` + 手机端 `<div class="mob-*">` 共存一份 HTML
- `<html data-layout="mobile" | "desktop">` 标记当前布局
- CSS 用 Tailwind `lg:` 断点切换可见性
- 所有数据修改（`state.categories` / `state.cards` / `state.theme`）经 `saveState()` → `renderAll()` 单向流；`renderAll()` 同时刷新两套 DOM
- 新增两个 state 字段：`view` ('hub' | 'categories' | 'archive') + `currentCategoryId` (string | null)，不参与 state-server 同步，仅内存态（默认 'hub' / null）

### 决策 2：手机端外壳（顶部栏 + 底部 nav + 搜索入口）

**顶部栏**（fixed top, h-14, `backdrop-blur-xl bg-surface-container/70`）
- 左槽：`#mobile-add-btn` 猫抓（add_circle 图标）→ 打开添加站点 sheet
- 中槽：`#mobile-title` 文字（跟随 view 变："枢纽" / 分类名 / "档案"）
- 右槽 1：`#mobile-theme-btn`（dark_mode / light_mode 切换）
- 右槽 2：`#mobile-search-btn`（search 图标）
- 安全区：`padding-top: env(safe-area-inset-top)`

**搜索入口**（Spotlight 模式，方案 A）
- 点搜索图标 → 顶部栏下方滑出搜索框（顶栏不消失，自动 focus）
- 占位符 "搜索站点或分类…"
- 输入即过滤当前视图的卡片（在 hub 搜全部 / 在 categories 搜当前分类 / 在 archive 搜分类名）
- 再点 X 收回

**底部导航**（fixed bottom, h-16, 三 tab 等宽）
- 左：`#tab-hub`（home 图标）→ `state.view = 'hub'`
- 中：`#tab-categories`（apps 图标）→ `state.view = 'categories'`, `currentCategoryId = null`（显示分类网格）
- 右：`#tab-archive`（inventory_2 图标）→ `state.view = 'archive'`
- 激活态：图标 `FILL 1` + `text-primary` + 上方 4px `bg-primary` 圆条
- 安全区：`padding-bottom: env(safe-area-inset-bottom)`
- 主区域切换时 `animate-stagger-fade-in` 重新入场（错落 30ms）

**子页导航**：
- 在 categories tab 点某分类 → `state.currentCategoryId = 'cat-foo'`，**子页**
- 子页时顶部栏左侧出现 `#mobile-back-btn` 返回箭头（仅 icon），按一下回到 `currentCategoryId = null`
- 子页时底部 tab 仍高亮"分类"

### 决策 3：卡片网格 + 弹窗

**卡片列表**（手机单列）
- 每张卡 `calc(100vw - 32px)` 宽、~96px 高
- 圆角 `rounded-2xl`、`bg-white/60 backdrop-blur-xl`
- 布局：左 56×56 图标 + 右标题（label-lg）/ 域名 · 描述（body-sm）
- 交互：
  - 单击 → 跳 URL（与桌面一致）
  - 长按 75ms → SortableJS 拖拽态（与桌面共用 `sortable-drag`）+ `navigator.vibrate(10)`
  - **右滑卡片（横滑距离 > 30px）→ 弹操作菜单**（**主要操作**，用户确认）
- 入场：`animate-stagger-fade-in`（30ms 错落，上限 8 张后归 0）
- **关掉** `animate-float`（手机端不浮动，避免列表滚动读不准）

**弹窗一：添加站点（Bottom Sheet）**
- 高度 `auto`（最大 92vh，键盘弹起时缩到 60vh）
- 拖拽手柄 24×4 居中、header h-14（标题 + X 关闭）、footer sticky bottom（取消 + 保存）
- 打开：300ms `cubic-bezier(0.16, 1, 0.3, 1)` 滑入 + 背景 `bg-scrim/40` 淡入
- 关闭：点背景 / 上滑手势 / X 按钮 → 200ms 滑出
- 上滑手势：监听 `touchmove` 在 sheet 顶 24px 区域 y 位移 > 80px 触发

**弹窗二：分类档案（管理分类，全屏 Sheet）**
- 占满 100vh（rounded-t-3xl 但延伸到顶部）
- 列表行：左侧色块圆点 + 分类名 + Material 图标 + 右侧 🗑 删除按钮（仅非 cat-other 显示）
- 长按 75ms → 拖拽重排
- "创建分类" 按钮点击 → 内联展开新行（色块选择器 + 名称输入 + 确认按钮），**不弹新弹窗**

**弹窗三：卡片操作菜单（Action Sheet）**
- 高度 ~40vh，4 行（编辑 / 移动到 / 删除 / 取消），每行 56px
- 顶部 h-14 小卡片预览（80% 不透明玻璃）
- 删除行文字红色
- "移动到" → 内容淡出 150ms → 分类列表淡入 200ms，顶部带返回箭头

**弹窗四：确认对话框（共用）**
- 高度 `auto`（通常 ~280px）
- imageUrl 参数决定顶部 img / span（图检索替换图标场景）
- 危险操作：主按钮改 `bg-error text-on-error`
- 普通：主按钮 `bg-primary`

**弹窗五：toast**
- 位置改为底部（nav 上方 16px），其余不变
- 维持 2.5s 自动消失

**z-index 栈**：
- 顶部栏 / 底部 nav / 桌面侧边栏：z-40
- sheet 背景遮罩 + sheet 内容：z-50
- toast：z-60
- 多个 sheet 不共存（开新关旧）

### 决策 4：CSS 策略

**断点**：Tailwind `lg:` (1024px) 硬切
- `< lg` → 手机端（启用 mob-*、显示手机外壳、隐藏桌面）
- `≥ lg` → 桌面端（启用 desk-*、显示桌面侧边栏、隐藏手机外壳）
- 不用 sm / md 切（平板 768px 走手机版更顺；1024px 是"手持→桌面"心理阈值）

**实现方式**：CSS only 切换（方案 A）
- HTML 同时包含两套 DOM
- CSS 用 `lg:hidden` / `hidden lg:block` 控可见
- JS 永远操作两套 DOM（renderAll 各跑一次）
- **不**用 JS 检测 innerWidth 动态插拔（避免闪烁、SortableJS 重建、滚动位置丢失）

**命名规范**：
- `.mob-*` → 仅手机端使用（`lg:hidden` 收尾）
- `.desk-*` → 仅桌面端使用（`hidden lg:block` 开头）
- `.shared-*` → 两端共用

**`isMobile()` + 类切换**：
```js
function isMobile() { return window.innerWidth < 1024; }
function cardClasses(colorKey) {
    const base = `shared-card glass-card glass-card-${colorKey}`;
    return isMobile()
        ? `${base} rounded-2xl p-4 flex flex-row gap-3 items-center`
        : `${base} rounded-[2rem] p-8 flex flex-col`;
}
```

**resize 处理**（节流 200ms）：
```js
let resizeTimer;
window.addEventListener('resize', () => {
    clearTimeout(resizeTimer);
    resizeTimer = setTimeout(() => {
        // 1) 销毁旧 SortableJS 实例
        // 2) applyMobileLayoutClass() 改 data-layout
        // 3) renderAll() 重建两套 DOM
        // 4) 创建新 SortableJS 实例
    }, 200);
});
```

**关键 CSS 规则**：
```css
@media (min-width: 1024px) {
    #main-content { padding-left: calc(var(--sidebar-width, 280px) + 60px); }
    #card-grid { grid-template-columns: repeat(auto-fill, minmax(280px, 1fr)); gap: 24px; }
    #mobile-topbar, #mobile-bottom-nav { display: none !important; }
}
@media (max-width: 1023.98px) {
    #desktop-sidebar, #desktop-topbar { display: none !important; }
    #card-grid { display: flex; flex-direction: column; gap: 12px; }
    #main-content { padding-top: 56px; padding-bottom: 72px; }
    body { -webkit-tap-highlight-color: transparent; }
    .glass-card { animation: none !important; }
}
```

**SortableJS 独立实例**（不跨端拖拽）：
- 手机端：`#card-grid-mobile` ↔ `#sidebar-categories-mobile` 共享 `group: 'mobile'`
- 桌面端：保留现有 `#card-grid` ↔ `#floating-sidebar` 共享 `group: 'shared'`
- 两套互不干扰

### 决策 5：保留复用清单 + 风险回滚

**数据层 / 持久化 / 启动三层 0 改动**

**JS 函数 ~80% 保留**：

**CSS / 样式**：

- 新增：mob-topbar / mob-bottom-nav / mob-search-overlay / mob-action-sheet / mob-bottom-sheet / mob-card / mob-tab-indicator / mob-saf-area-* / mob-sheet-handle + 媒体查询
- 桌面 .floating-sidebar / .glass-card / .magic-circle-bg / .incantation-modal / .brand-mark / .hero-panel / .search-shell / .filter-panel 全部不动

**风险清单**：

| 风险 | 等级 | 应对 |
|------|------|------|
| 双 DOM 数据不一致 | 🟡 中 | 任何 state 修改走 `saveState` → `renderAll` 单向流 |
| SortableJS 实例泄露 | 🟡 中 | resize 时先 destroy 旧实例再创建新实例 |
| 跨断点闪烁 | 🟢 低 | CSS only 切换天然不闪烁 |
| iOS 100vh bug | 🟡 中 | 用 `100dvh` + `-webkit-fill-available` fallback |
| iOS 点击高亮 | 🟢 低 | `-webkit-tap-highlight-color: transparent` |
| iOS 长按系统菜单 | 🟡 中 | 卡片 `touch-callout: none` + `user-select: none` |
| 安卓低端机动画掉帧 | 🟡 中 | `prefers-reduced-motion` + IntersectionObserver stagger |
| 备份文件被误 commit | 🟢 低 | `.bak` 已加到 .gitignore |

**回滚方案**：
```bash
cp 链接跳转.pre-mobile-redesign-2026-06-11.html.bak 链接跳转.html
# 重启 .cmd
```
回滚不影响 localStorage / state.json / 默认数据。
